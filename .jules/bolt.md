## 2025-02-14 - O(N log N) bottlenecks in data transformations
**Learning:** Extracting the "latest" item from grouped database results using chained `.filter().sort().reduce()` creates a severe `O(N log N)` computational bottleneck and unnecessary object allocations during API response mapping, specifically noticed in Deals mapping in `route.ts` and `_lib.ts`.
**Action:** When grouping tabular data in JavaScript and picking a single row based on a condition (like the latest `created_at`), always initialize local `Map` stores and use a single-pass `for...of` loop to compare and store the optimal row, dropping `O(N log N)` sorts down to strict `O(N)`.
